Conclusion et Recommandations

Jawad SAJID, Mael BERTHELOT, Noan KOUKA et Maxime MARC

Bilan pour l'entreprise KDou

Notre analyse a révélé que la principale faiblesse de KDOU réside dans la fiabilité de ses données. Des problèmes récurrents (fautes de frappe, données manquantes, manque de standardisation) compliquent les analyses et peuvent fausser les indicateurs de performance.

Pour que KDOU puisse se baser sur ses données pour prendre des décisions stratégiques (notamment sur son virage éco-responsable), une phase de nettoyage et de normalisation est une étape indispensable. L'adoption d'une structure de base de données relationnelle stricte est la clé pour garantir l'intégrité des informations à long terme.

Bilan sur les outils utilisés

Excel s'est révélé être un outil inadapté pour la gestion d'une base de données de cette envergure. Il favorise les erreurs de saisie et ne garantit pas l'intégrité des données (clés, relations).

Microsoft Access, en revanche, est un outil bien plus puissant pour cet usage. Il permet de définir un schéma relationnel et offre deux modes d'interrogation complémentaires :

  • Le QBE, très efficace pour des requêtes visuelles rapides.
  • Le SQL, indispensable pour toute analyse complexe et pour la création d'indicateurs de performance précis.
