Mission 2 : Exploitation de la Base

Jawad SAJID, Mael BERTHELOT, Noan KOUKA et Maxime MARC

Contexte

Pour cette deuxième mission, nous avons migré les données d'Excel vers Access. Cet outil a permis de structurer la base de données de manière plus robuste, notamment en définissant des relations claires entre les tables. L'objectif était de commencer à exploiter ces données via des requêtes, en utilisant à la fois l'interface graphique QBE et le langage SQL.

Indicateurs de la base de données

Premier indicateur : Existe-t-il des fournisseurs sans produit ?

Cette requête vise à nettoyer le catalogue de fournisseurs. Identifier ceux qui n'ont aucun produit référencé permet à KDOU de faire un choix stratégique : soit les relancer pour obtenir leur catalogue, soit les supprimer de la base pour optimiser la gestion des partenaires et ne pas fausser les statistiques.

Code SQL de la requête :

SELECT DISTINCT Fournisseur.*
FROM Fournisseur
WHERE Fournisseur.NoFour NOT IN
( SELECT Produit.NoFour FROM Produit );

Deuxième indicateur : Clients n'ayant pas encore passé de commande

Identifier les clients "inactifs" est crucial pour l'action commerciale. Cette liste permet à l'équipe de KDOU de cibler ces clients avec des offres de bienvenue ou des campagnes de démarchage spécifiques pour générer de nouvelles ventes et améliorer la rentabilité.

Code SQL de la requête :

SELECT Cli.CodeCli, Cli.Societe, Cli.MailCli
FROM Clientele AS Cli
LEFT JOIN Commande AS Com ON Cli.CodeCli = Com.CodeCli
WHERE Com.CodeCli IS NULL
ORDER BY Cli.Societe;
