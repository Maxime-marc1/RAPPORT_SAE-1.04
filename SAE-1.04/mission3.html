Mission 3 : Indicateurs Avancés (SQL)

Jawad SAJID, Mael BERTHELOT, Noan KOUKA et Maxime MARC

Contexte

La dernière mission s'est concentrée sur des analyses plus complexes en utilisant du SQL avancé. Nous avons travaillé sur une nouvelle version de la base de données, enrichie et mieux structurée (par exemple, avec l'ajout d'une table `Pays` dédiée, améliorant la normalisation et la fiabilité des requêtes géographiques). L'interface QBE n'étant plus suffisante pour ces indicateurs calculés, le SQL pur était nécessaire.

Indicateurs intéressants de la base de données

(Note : Conformément aux consignes, voici les indicateurs proposés par les membres de l'équipe.)

Indicateur 1 : Optimisation logistique et circuit-court

Objectif : Quels sont les fournisseurs et clients qui se trouvent dans la même ville ?

L'intérêt de cette requête est double. D'une part, elle permet une optimisation logistique en identifiant des opportunités de regroupement de transports. D'autre part, elle répond directement à l'objectif d'éco-responsabilité de KDOU : en favorisant un circuit court (fournisseur local vers client local), l'entreprise peut activement réduire son empreinte carbone liée aux livraisons.

Code SQL :

SELECT DISTINCT
Fournisseur.Societe AS Fournisseur,
Clientele.Societe AS Client,
Fournisseur.Ville AS Ville
FROM
( Fournisseur
INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour )
INNER JOIN ( ( Cliente INNER JOIN Commande ON Clientele.CodeCli = Commande.CodeCli )
INNER JOIN DetailCommande ON Commande.[N° commande] = DetailCommande.[N° commande] ) ON Produit.RefProd = DetailCommande.[Réf produit]
WHERE
Fournisseur.Ville = Clientele.Ville
AND Fournisseur.NoPays = Clientele.NoPays

Indicateur 2 : Sourcing et approvisionnement européen

Objectif : Lister tous les produits provenant de pays membres de l'Union Européenne.

Cette requête aide KDOU à valoriser son image "éco-friendly". En identifiant facilement les produits d'origine UE, l'entreprise peut les mettre en avant dans son marketing, un argument fort pour les clients sensibles à l'empreinte carbone et à l'économie locale (européenne).

Code SQL :

SELECT
Pays.PaysFR AS Pays,
Produit.NomProd AS Produit,
Categorie.NomCateg AS Catégorie,
Fournisseur.Societe AS Fournisseur
FROM
((Produit
INNER JOIN Categorie ON Produit.CodeCateg = Categorie.CodeCateg)
INNER JOIN Fournisseur ON Produit.NoFour = Fournisseur.NoFour)
INNER JOIN Pays ON Fournisseur.NoPays = Pays.NoPays
WHERE
Pays.UE = "Membre"
ORDER BY
Pays.PaysFR,
Produit.NomProd;
